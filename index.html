Overview

Decentra is a powerful, self-hosted alternative to Discord. Built with Python and WebSockets, it offers real-time messaging, voice chat, and persistent data storage using PostgreSQL. Perfect for individuals and communities who want complete control over their chat infrastructure.

üöÄ Quick Start

Prerequisites: Docker and Docker Compose installed on your system.

Step 1: Clone the Repository

git clone https://github.com/SluberskiHomeLab/decentra.git
cd decentra

Step 2: Configure Environment

cp .env.example .env

Edit .env and update the database credentials:

# PostgreSQL Database Configuration
POSTGRES_DB=decentra
POSTGRES_USER=decentra
POSTGRES_PASSWORD=your_secure_password_here

# Server Configuration
DATABASE_URL=postgresql://decentra:your_secure_password_here@postgres:5432/decentra

Step 3: Start the Server

docker-compose up --build

Step 4: Access the Application

Open your web browser and navigate to:

http://localhost:8765

Create an account or log in to start chatting! The first user can sign up without an invite code.

Managing Data

Stop the server (data persists):

docker-compose down

Remove all data and start fresh:

docker-compose down -v
Note: Your data is stored in a PostgreSQL database in the Docker volume named decentra-data and will persist across container restarts.

üèóÔ∏è Architecture

Server Layer

  • server.py: Combined HTTP and WebSocket server
  • database.py: PostgreSQL data layer
  • api.py: REST API endpoints
  • Handles authentication, messaging, and real-time communication

Database

  • PostgreSQL 16: Production-ready relational database
  • Stores users, servers, channels, messages, and friendships
  • Docker volume for data persistence
  • Automatic schema initialization

Web Client

  • index.html: Login and signup page
  • chat.html: Main chat interface
  • voice.js: WebRTC voice implementation
  • Discord-like responsive UI design

Technology Stack

  • Backend: Python with WebSockets and aiohttp
  • Database: PostgreSQL 16 (via psycopg2-binary)
  • Voice: WebRTC for peer-to-peer audio
  • Security: bcrypt for password hashing
  • Deployment: Docker and Docker Compose

üìñ Usage Guide

Creating Servers

Click the + button in the left sidebar to create a new server. Each server automatically gets a "general" channel. Server owners can:

  • Rename the server
  • Generate invite codes
  • Manage user permissions (create/edit/delete channels)

Managing Friends

  1. Click the Friends button in the header
  2. Click Search Users to find other users
  3. Click Add Friend to connect instantly
  4. Start private conversations via DMs

Voice Chat

Direct Calls: Click the üìû button next to a friend's name to start a voice call.

Voice Channels: Users with permissions can create voice channels in servers. Click on a voice channel (üîä icon) to join.

  • Use the mute button (üé§) to control your microphone
  • See how many users are in each voice channel
  • Click "Leave Voice" to disconnect

Permissions System

Server owners can manage granular permissions for each member:

  • Create Channel: Allow creating new text/voice channels
  • Edit Channel: Allow modifying channel settings
  • Delete Channel: Allow removing channels
